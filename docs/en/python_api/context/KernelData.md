# KernelData<a name="ZH-CN_TOPIC_0000002154810613"></a>

KernelData is the structure used for [KernelMonitor.start](KernelMonitor-start.md) calls and is defined as follows:

```python
class KernelData:
    self.kind  # Activity Record type MSPTI_ACTIVITY_KIND_KERNEL
    self.start   # Start timestamp of kernel execution on the NPU device, in ns. If both the start and end timestamps are 0, the kernel timestamp information cannot be collected
    self.end   # End timestamp of kernel execution, in ns. If both the start and end timestamps are 0, the kernel timestamp information cannot be collected
    self.device_id   # Device ID of the device where the kernel runs
    self.stream_id   # Stream ID of the stream where the kernel runs
    self.correlation_id   # Unique ID generated by the runtime when launching the kernel. Other activities can associate with the kernel through this value
    self.type    # Type of the kernel
    self.name   # Name of the kernel. This name remains consistent throughout the activity record and is not recommended to be modified
```
